# Dynamic Serverless Energy Dashboard - Implementation Summary

## Overview

Transformed the Grafana dashboard and simulator from a hardcoded system to a fully dynamic component discovery system that automatically adapts to any number of serverless components.

## What Changed

### Before (Hardcoded)
- **simulator.py**: Explicitly named 5 Lambda functions, 2 S3 buckets, 2 DynamoDB tables, 1 API, 1 Rekognition service
- **dashboard.json**: All Flux queries hardcoded with specific component names
- **Adding components**: Required editing code and regenerating dashboard manually

### After (Dynamic)
- **simulator.py**: Automatically discovers all .xlsx files in category folders
- **dashboard.json**: Regenerated by script based on discovered components
- **Adding components**: Simply add Excel file to folder, run regeneration script

## Key Achievements

### 1. Dynamic File Discovery (simulator.py)
```python
# Old: METRIC_FILES = { "Lambda": {"DoorbellLambda": path, "FaceRecognitionLambda": path, ...}}
# New: METRIC_FILES = discover_metric_files()  # Finds all .xlsx files dynamically
```

**Changes**:
- Replaced hardcoded `METRIC_FILES` dictionary with `discover_metric_files()` function
- Uses `Path.glob("*.xlsx")` to find all files in each category folder
- Component name derived from filename without extension
- Automatically scales to any number of components

### 2. Dashboard Generation System

Created two new tools:

#### **generate_dashboard_complete.py** (Main tool)
- Loads existing dashboard.json
- Generates Flux queries dynamically for all discovered components
- Updates all 8 panels with new queries
- Preserves dashboard layout, colors, thresholds

#### **generate_dashboard.py** (Preview tool)
- Generates queries without modifying dashboard
- Useful for testing and validation

### 3. Query Structure

**Old Query** (for Lambda):
```flux
ELcompute: (((if exists r["lambda_DoorbellLambda_Tcpu"] then ... ) + 
            ((if exists r["lambda_FaceRecognitionLambda_Tcpu"] then ... ) + ...
            [hardcoded for each function]
```

**New Query** (adaptive):
```python
# Python generates:
lambda_tcpu_malloc = "(component1_calc) + (component2_calc) + ... + (componentN_calc)"
# Applied to Flux query dynamically
```

## Usage Workflow

### Adding a New Component
1. Add new Excel file to appropriate folder:
   ```bash
   cp NewLambdaFunction.xlsx Final Serverless Metrics/Lambda/
   ```

2. Run simulator (auto-discovers new file):
   ```bash
   python3 simulator.py
   ```

3. Regenerate dashboard (includes new component in queries):
   ```bash
   python3 generate_dashboard_complete.py
   ```

4. Reload Grafana dashboard in browser

### Verification
Run system tests to verify everything works:
```bash
python3 test_system.py  # All 5 tests should pass
```

## Technical Details

### Query Generation Algorithm

For each category (Lambda, S3, etc.):
1. Discover all component files: `glob("*.xlsx")`
2. Extract component names from filenames
3. For each metric, generate aggregation:
   - Lambda compute: `(Tcpu1 × Malloc1) + (Tcpu2 × Malloc2) + ...`
   - S3 buckets: `B1 + B2 + ... + Bn`
   - etc.
4. Apply energy formulas to aggregations
5. Generate complete Flux query

### Field Naming Convention

All fields follow pattern: `{category}_{component_name}_{metric}`

Examples:
- `lambda_DoorbellLambda_Tcpu` → Lambda DoorbellLambda CPU time
- `s3_mhm-home-security-img_B` → S3 bucket size
- `dynamodb_Events_Lsuccess` → DynamoDB Events table requests
- `api_API_LAPI` → API Gateway latency
- `rekognition_Rekognition_Tresponse` → Rekognition response time

### Energy Formulas (Unchanged)

All formulas remain identical to energy_calculator.py:

- **Lambda Compute**: `(Tcpu × Malloc × P_core) / (1000 × 3600 × 1000 × Mref)`
- **Lambda Memory**: `(Mused × P_mem × (Texec + Tinit)) / (1024 × 1000 × 3600 × 1000)`
- **Lambda Storage**: `(Sused × P_storage × (Texec + Tinit)) / (1024³ × 1000 × 3600 × 1000)`
- **Lambda Network**: `(Dnetwork × E_network_intensity) / 1024³`
- **S3 Active**: `Σ(Rtotal_i × Dlatency_i) × P_active / (1000 × 3600 × 1000)`
- **S3 Idle**: `((ΣB_i + N_inv × S_upload) / 1024³) × (P_idle / (C_mem × 1000))`
- **DynamoDB**: `ΣLsuccess_i × P_active / (1000 × 3600 × 1000)`
- **API Gateway**: `ΣLAPI_i × P_proc / (1000 × 3600 × 1000)`
- **Rekognition**: `ΣTresponse_i × P_GPU / (1000 × 3600 × 1000)`
